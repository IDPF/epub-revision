About this vocabulary

This vocabulary, a part of EPUB 3.0, defines a set of metadata properties for use in [../../spec/epub30-publications.html#sec-package-documents Package Documents].

Package-level Metadata

content-version

The {content-version} property provides the version number of the publication. The semantics of the three digit sequences and how they are incremented for any given publication is not defined.

publication-type

The {publication-type} property describes the nature of a specialized type of EPUB Package. The value {dictionary} indicates that the package contains a dictionary as defined in TODO and the value {annotation} that the package contains an annotation as defined in TODO.

source-identifier

The {source-identifier} property references the {identifier} (element or property) that is the source of the NCX pageList (i.e., the publication from which the pagination was derived).

page-progression-direction

The {page-progression-direction} property indicates the global direction in which the publication content flows. This property can be overriden by individual manifest {items}s.

Ancillary metadata properties

alternate-script

The {alternate-script} property provides an alternate expression of the associated property value in a language and script identified by the {xml:lang} attribute. The property is typically attached to {creator} and {title} properties for internationalization purposes.

date-ref

The {date-ref} property relates an { event} to the {date} on which it occurred. This property is typically used to tie changes to metadata to the corresponding revision date for the publication. !Why was this under metadata authority props? Is it global?^

display-seq

The {display-seq} property indicates the numeric position in which to display the current property relative to identical metadata properties. For example, to indicate the order in which to render multiple {title} elements or properties.

event

The {event} property indicates a significant event in the lifecycle of a publication. Events are typically modify {date} elements or properties to provide the reason for a change to the publication. A set of standard values are defined for this property but no restrictions are imposed on the extension of this list.

file-as

The {file-as} property provides the normalized form of the associated property for sorting.

role

The {role} property describes the nature of work performed by a {creator} or {contributor}, for example that they represent the author or editor of a publication.

scheme

The {scheme} property identifies the formal system used to express the value of an {identifier} or {subject} element or property.

title-order

The {title-order} property indicates the numeric position in which to sort the associated {title} relative to other titles identified with the same { title-type} value. !was sort-order, but unclear if any other use for the property with display-seq, and whether this is already covered by file-as.^

title-type

The {title-type} property indicates the form or nature of a {title}. A set of standard values are defined for this property but no restrictions are imposed on the extension of this list. !isn't this a role?^

Metadata Authority properties

authority

The {authority} property references the metadata authority ({ meta-auth}) responsible for an instance of package metadata.

meta-auth

The {meta-auth} property provides the name of a party or authority responsible for the package metadata.

signature

The {signature} property relates a metadata authority ({ meta-auth}) responsible for creating or updating package metadata to an XML Signature contained in the Signatures.xml file.

Manifest properties

cover-image

The {cover-image} property identifies a resource as the cover image for the publication.

mathml

The {mathml} property indicates that the associated resource contains one or more instances of MathML markup.

online-resources

The {online-resources} property indicates that the associated resource references publication resources that are located online.

page-spread-left

The {page-spread-left} property indicates left-to-right directionality of a top-level SVG document. This property must always be used in conjunction with the { svg} property when describing a manifest {item}. It must not be used simultaneously with the { page-spread-right} property.

page-spread-right

The {page-spread-right} property indicates right-to-left directionality of a top-level SVG document. This property must always be used in conjunction with the { svg} property when describing a manifest {item}. It must not be used simultaneously with the { page-spread-left} property.

scripted

The {scripted} property indicates that the associated resource contains scripted content.

svg

The {svg} property indicates that the associated resource contains one or more instances of SVG markup.

Metadata record types

onix-record

The {onix-record} property indicates the referenced resource is an ONIX record.

mods-record

The {mods-record} property indicates the referenced resource is a MODS record.

xmp-record

The {xmp-record} property indicates the referenced resource is an XMP record.

marc21xml-record

The {marc21xml-record} property indicates the referenced resource is a MARC21 record.
