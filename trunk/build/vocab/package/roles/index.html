Overview

This document lists all IDPF-defined roles for use with the Package Document collection element.

Roles are typically defined in IDPF sub-specifications, in which case a pointer to the official definition is provided with the role name, but this document also contains normative definitions for general-purpose roles not tied to a specific specification.

This document is not tied to a particular version of the EPUB® specification.

Conformance Statements

The keywords MUST, MUST NOT, REQUIRED, SHALL, SHALL NOT, SHOULD, SHOULD NOT, RECOMMENDED, MAY, and OPTIONAL in this document are to be interpreted as described in [RFC2119].

All role definitions in this document are normative.

All examples are informative.

Revision Policy

This document is subject to change at any time as new roles are defined by IDPF Working Groups. The roles defined herein will never be removed, but may be deprecated.

Roles

manifest

Manifest sub-collections are included in collection elements that identify a set of related EPUB Content Documents, but where processing of those documents is dependent on knowing all of the resources they reference. For example, a preview might identify the XHTML Content Documents containing the sample chapter(s), but a Reading System will also need to know the style sheets, images and other resources used in rendering in order to effectively extract them for viewing.

When a collection element’s role attribute specifies the term "manifest", it indicates that the link element children of the collection represent such a set of Publication resources.

A manifest collection must contain only one or more link elements, where the IRI contained in each link element’s href attribute must reference the location of a required resource. The manifest collection should include a complete set of resources, including all Content Documents listed in the parent.

A manifest collection must be the child of another collection.

Implementations of the collection element that include a manifest may set restrictions on the types of resources that can be referenced.
