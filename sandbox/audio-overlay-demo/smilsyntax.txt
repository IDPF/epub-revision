About the SMIL elements and attributes used:

seq: a sequence of media nodes
	@role
	@id
	@textref
	allowed children: seq, par

par: a parallel set of media nodes
	@role
	@id
	allowed children: text, audio
	
text: references text media
	@src
	allowed children: none
	
audio: references audio media
	@src
	@clipBegin
	@clipEnd
	allowed children: none

smil
	@textdoc
	allowed children: one seq
	
Rules

 # 1 SMIL file per HTML file
 # Seq nodes have @textref pointing to the relevant text structure
 # Par nodes contain one text and one audio
 # the text and audio inside a par are considered equivalent to each other content-wise
 # the html filename goes in smil@textdoc
 # when the text@src and seq@textref values are just #frag with no filename, they should be resolved against smil@textdoc
